## 2024-05-22 - [Optimizing Number Format Checks and Constant Lookups]
**Learning:** Core functions like `is_date_format` and `is_builtin` in `openpyxl/styles/numbers.py` are major bottlenecks when processing large spreadsheets. `is_builtin` was performing O(n) lookups on `BUILTIN_FORMATS.values()`, which can be optimized to O(1) by using the existing `BUILTIN_FORMATS_REVERSE` dictionary. Regex-heavy functions like `is_date_format` benefit enormously (~15x) from `functools.lru_cache`. Additionally, membership checks on tuples like `ERROR_CODES` in `openpyxl/cell/cell.py` should be converted to `frozenset` for O(1) performance.
**Action:** Apply `lru_cache` to repeated regex checks and ensure constant-time lookups for frequently used membership tests. Always use `PYTHONDONTWRITEBYTECODE=1` to avoid touching tracked `__pycache__` files in this repository.
